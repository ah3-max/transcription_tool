## 1. 問題定義

### 1.1 現狀痛點
- 長照機構的會議與護理交班以口說為主，事後要靠人手把錄音整理成逐字稿，一場 30–60 分鐘的會議往往要花數倍時間聽打。
- 現場語言以**中文為主、夾雜英文**；同時有泰籍同仁，需要把內容轉成**泰文**才能傳達，目前無即時管道。
- 資料涉及機構內部營運與潛在個資（住民、護理內容），**不可外送雲端**處理。

### 1.2 目標
- 即時將中文（含中英夾雜）語音轉文字並翻譯為泰文／英文，延遲控制在「幾句話」之內。
- 批次將既有錄音檔轉成乾淨逐字稿與翻譯，並依範本產生會議記錄／護理交班文件。
- 全程本地推論、零雲端依賴，作為未來「護理語音紀錄／交班」工作流的前導練習。

### 1.3 成功指標
| 指標 | 目標 | 量測方式 |
|---|---|---|
| 中文逐字稿可用度 | 真實會議抽樣，字錯誤率低到「免大改即可用」 | 人工抽樣比對 |
| 中→泰可讀性 | 泰語母語者評為「可讀、達意」 | 母語者評分 |
| 即時延遲 | 譯文落後語句邊界約 1–3 句 | 計時觀察 |
| 共用 VM 友善度 | 不超出動態額度、閒置可釋放顯卡 | 監看 VRAM/RAM |

## 2. 使用者與角色

### 2.1 角色定義
| 角色 | 說明 | 人數 | 使用頻率 |
|---|---|---|---|
| 操作者（唯一角色） | 機構內技術／行政人員，本人即開發者與維運者 | 1–少數 | 不定期 |

> Phase 1 為單一／少數人內網測試，**無登入、無權限分層**。資安仍須處理（見 §5）。多人或正式部署再導入角色（見開發執行手冊安全故事）。

### 2.2 權限矩陣
| 操作 | 操作者 |
|---|---|
| 即時翻譯、批次轉錄、文件生成、設定模型路由、調整保留與資源策略 | ✓（全功能） |

### 2.3 使用情境
- **情境 A（即時交班）**：交班時打開即時翻譯，選好麥克風與「輸出泰文」，畫面大字顯示泰文譯文、小字顯示中文原文，泰籍同仁邊聽邊看；結束後該場自動存入錄音記錄，可再產生摘要或交班文件。
- **情境 B（會議事後整理）**：把錄好的週會 m4a 丟進批次逐字稿，選輸出中＋泰，系統切段轉錄並翻譯，產出乾淨逐字稿，再到文件生成套「會議記錄」範本匯出 docx。
- **情境 C（共用 VM 同事也在用）**：操作者啟動批次工作時，系統只取用當下可用資源的有限定額，跑完或閒置即釋放顯卡，不影響同台其他專案。

## 3. 範圍與非目標

### 3.1 Phase 1 範圍
即時翻譯、錄音記錄、批次逐字稿、文件生成（會議記錄／護理交班／自訂範本）、模型路由設定、資源自限與閒置釋放、介面多語與深色模式。

### 3.2 非目標（明確不做）
| ID | 非目標 | 理由 |
|---|---|---|
| NG-1 | 不上雲、不接外部 API | 隱私／合規；全本地是核心前提 |
| NG-2 | Phase 1 不做登入／權限系統 | 單一／少數人內網測試 |
| NG-3 | 不輸出 PDF | PDF 牽涉 HTML 轉檔等，穩定度不足；僅 docx／md／txt |
| NG-4 | 不支援台語 | 非主力且模型支援未證實 |

| NG-6 | 即時不保證逐字級時間戳 | 串流模式不回時間戳 |
| NG-7 | 離線端（Mac/iPhone）不跑 31B | 18GB 統一記憶體不可行，離線僅轉錄或小模型 |

## 4. 功能需求（FR）

### 4.1 即時翻譯
- **FR-1** 可選麥克風來源；按開始後進入專屬即時畫面。
- **FR-2** 輸入語言可選中文／中文＋英文／純英文；輸出翻譯語言可複選（泰／英／中）。
- **FR-3** 即時畫面以單一主要輸出語言放大顯示，保留中文原文小字，可切換顯示語言。
- **FR-4** 字級放大／縮小時主譯文與原文等比例縮放。
- **FR-5** 串流同時持續錄音；停止後存入錄音記錄。
- **FR-6** 以語句／靜音邊界（VAD）切段；延遲容忍「幾句話」。
- **FR-7** 可隱藏／顯示辨識原文側欄。

### 4.2 錄音記錄
- **FR-8** 列出每場錄音（名稱、時間、長度、狀態），保留 7 天。
- **FR-9** 開啟單場可看逐字稿與摘要；摘要按需產生，產生時顯示「統整中」緩衝。
- **FR-10** 可重新翻譯、匯出（docx／md／txt）、下載原始音檔。

### 4.3 批次逐字稿
- **FR-11** 可切「新增轉入／歷史記錄」。
- **FR-12** 多檔上傳（mp3／mp4／m4a／wav）、序列處理；每 15 分鐘自動切段。
- **FR-13** 選項：辨識語言；輸出範圍多選（中／英／泰）。
- **FR-14** 雙層進度（整體第幾／共幾＋當前檔），進度區可收折。
- **FR-15** 結果以文件樣式預覽，可切中／泰頁與輸出格式（文件／Markdown／.txt）。
- **FR-16** 歷史記錄列出過往轉入與可下載產出。

### 4.4 文件生成
- **FR-17** 來源可「選擇來源」（錄音記錄／批次逐字稿產出，分類下拉）或「手動上傳」逐字稿檔。
- **FR-18** 範本選一般會議記錄／護理交班／上傳自訂範本。
- **FR-19** 切換範本時輸出預覽標籤與內文結構同步更新（會議記錄欄位 ≠ 交班欄位）。
- **FR-20** 輸出預覽以文件樣式呈現，可選格式匯出（docx／md／txt）。

### 4.5 通用／設定
- **FR-21** 模型路由：ASR／批次翻譯／即時翻譯／後處理摘要各自可選模型、可新增端點。
- **FR-22** 新增端點表單：名稱、服務網址、模型名稱、處理功能。
- **FR-23** 介面語言中／英／泰切換（介面在地化；使用者內容不自動翻譯）。
- **FR-24** 右上資源用量（VRAM／RAM 百分比）**預設隱藏**、可開；深色／淺色可切換。
- **FR-25** 匯出一律「選格式下拉＋匯出鈕」，僅 docx／md／txt。

## 5. 非功能需求（NFR）

### 5.1 一般
- **NFR-1（資源）** 動態取用、不一次全佔：啟動工作查當下可用量、reserve 上限定額，閒置逾時整包釋放、下次重載。
- **NFR-2（降級）** 無法配置即時翻譯資源時，提醒改手動錄音並停止即時服務。
- **NFR-3（儲存）** 在 2TB 磁碟（2026-06-26 實測可用 ~1.3TB）內精簡使用；原始音檔壓縮落地；7 天到期自動清。
- **NFR-4（本地／內網）** 全部推論在本機；應用本體對**內網全網段**開放（Port 3600），模型端點僅 localhost，無對外（網際網路）連線需求。
- **NFR-5（無障礙，優先生效）** 內文最小 19–20px、行高 1.8、可點元件 ≥48×48px、對比 WCAG AA ≥4.5:1。
- **NFR-6（在地化）** 介面字串以鍵值管理，較長語言預留版面。
- **NFR-7（可維護）** 單一語言棧（Python/FastAPI）優先；批次與即時僅在 API/資料層相接。

### 5.2 資安需求（Security NFR）
> 即使全內網、單人測試，程式漏洞與資料風險仍須納入。對應開發執行手冊的安全故事與驗收。

- **SEC-1（供應鏈）** 模型／套件僅自官方來源取得，下載後比對雜湊；鎖版並定期 SCA 掃描 CVE。
- **SEC-2（輸入驗證）** 上傳嚴格白名單副檔名＋MIME＋實際解碼驗證；檔案大小／時長硬上限。
- **SEC-3（路徑安全）** 兩區儲存一律以伺服器產生的 id 組路徑，不用使用者檔名拼接；正規化並限制在 /data 內，防路徑穿越。
- **SEC-4（提示注入）** 後處理 LLM 會遵循指令，須把逐字稿／使用者文字當資料、與系統指令分離。
- **SEC-5（資源耗盡）** 序列處理＋佇列上限＋單檔上限＋動態預算，防單檔或併發拖垮共用 VM。
- **SEC-6（連線加固）** WebSocket 限同機／內網來源、限連線數與訊息大小、閒置斷線。
- **SEC-7（落地清除）** 7 天到期確實刪檔＋刪索引；暫存與日誌不長期保留；原始音檔考慮加密落地。
- **SEC-8（隔離／日誌）** /data 目錄權限收斂；日誌不記錄音內容／個資原文；錯誤訊息不外露路徑與堆疊。
- **SEC-9（部署擴大）** 轉多人或正式部署前導入登入與角色、網路僅綁內網介面。
- **SEC-10（個資／醫療）** 護理內容高敏感；建立去識別化與最小留存，Phase 3 前做合規檢視。

## 6. 假設（Assumptions，可隨時更正）
- A-1：Phase 1 單一／少數操作者、內網、無登入。
- A-2：Gemma 4 31B Q8（約 33GB）跑在 96GB GPU；即時翻譯預設 NLLB-3.3B。
- A-3：原始音檔壓縮落地，含即時錄音檔，保留 7 天。
- A-4：介面在地化涵蓋框架字串；範例／使用者內容維持原語言。
- A-5：輸出僅 docx／md／txt。

## 7. 待拍板（Open Questions）
- **OQ-1** 中→泰品質若候選模型皆不達標的退路（人工術語表／後編輯／縮小場景）。
- **OQ-2（已執行，2026-06-26 實測）** 原建議補 RAM（→64–128GB）與儲存（→1–2TB）：RAM 已升至 ~60 GiB（64GB 級，落在建議範圍下緣，未來若 RAM 吃緊可再加到 128GB）、儲存已達 2TB；VRAM 96GB 仍充裕。
- **OQ-3** 護理交班範本確切欄位（Phase 3 前定案）。
- **OQ-4** 即時延遲上限數值化驗收（目前以「幾句話」為準）。
