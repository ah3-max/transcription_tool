## 1. 技術選型

### 1.1 技術棧
| 層級 | 選擇 | 理由 |
|---|---|---|
| 後端框架 | Python + FastAPI | REST（批次）＋WebSocket（即時）一套；單語言棧好維護 |
| 任務佇列 | 內建背景工作（asyncio／簡易序列佇列） | 單機、序列處理即可，先不導入 Celery |
| 前端 | **原生 HTML/CSS/JS（直接沿用 v6 版面與 token）** | 內部工具、單人維護；同源相對路徑、無 build／烤環境變數問題 |
| UI 樣式 | 設計權杖 v4 的 CSS 變數（直接取自雛形） | 已有 token，不引框架與重型 UI 庫 |
| 模型服務 | 由 LM Studio／vLLM／提供商 pod 服務（OpenAI 相容端點，各自埠） | **ASR 即時串流必須 vLLM 或支援 ASR 的 pod；LM Studio 不支援 ASR**；文字 LLM 三者皆可 |
| ASR | Qwen3-ASR 1.7B（備援 0.6B／Whisper Large V3 Turbo） | 串流＋離線同一顆 |
| 前處理 | DeepFilterNet3、Silero VAD | 降噪勿過頭、以 VAD 切自然邊界 |
| 翻譯 | 即時 NLLB-3.3B；批次 Gemma 4 31B Q8／TranslateGemma 27B／NLLB | 路由可換、解耦扇出 |
| 後處理／摘要／文件 | Gemma 4 31B Q8 | 標點、摘要、會議記錄、術語修正 |
| 資料庫 | SQLite | 索引與工作狀態，免架 DB server |
| 文件產出 | python-docx（docx）、純文字（md/txt） | 不做 PDF |
| 部署 | 本機程序／docker compose | 應用本體綁 0.0.0.0 對內網開放（開發 3600／測試環境 3610）；對外不開 |

> 已確認約束：Gemma 4 31B Q8 ≈ 33GB（96GB 內可行）；**實測硬體（2026-06-26）：RAM ~60 GiB（64GB 級）、VRAM 96GB（RTX PRO 6000 Blackwell）、8 核 Xeon 6515P、磁碟 2TB；原「RAM 32GB 瓶頸」為升級前舊值**——RAM 已較寬裕，但仍是相對較小的池、且為共用 VM（啟動時 swap 已用 7.2GB／GPU 已佔 22.7GB），工作須走動態 reserve＋閒置釋放並設單檔／佇列上限防耗盡；Qwen3-ASR 串流模式不回時間戳、單段 ≤20 分鐘（批次每 15 分鐘切段）。

### 1.2 專案結構
```
stt-translate/
├── server/                     # 後端（FastAPI）
│   ├── main.py                 # app 進入點、路由掛載、內網綁定
│   ├── routes/                 # REST 路由：jobs / sessions / records / endpoints / resources
│   ├── ws/                     # WebSocket：live 串流
│   ├── services/               # 商業邏輯
│   │   ├── asr.py              # Qwen3-ASR 串流＋離線
│   │   ├── preprocess.py       # DeepFilterNet3 + Silero VAD
│   │   ├── translate.py        # 扇出翻譯、路由分派
│   │   ├── postprocess.py      # 摘要 / 會議記錄 / 交班（Gemma 4）
│   │   ├── export.py           # docx / md / txt 產出
│   │   └── resources.py        # 動態 reserve / 閒置釋放 / 降級
│   ├── models_db/              # SQLite schema 與存取
│   ├── storage/                # 兩區路徑管理、id 命名、清除排程
│   ├── middleware/             # 輸入驗證、大小上限、錯誤處理、日誌
│   └── config.py               # 端點、上限、保留天數等設定
├── web/                        # 前端（原生 HTML/CSS/JS，FastAPI 靜態服務）
│   ├── index.html              # 沿用 v6 愛愛院版版面（5 畫面）
│   ├── styles/tokens.css       # 設計權杖 v4
│   ├── app.js                  # 原生 JS：API 相對路徑、WS、主題、i18n 套用
│   ├── i18n.js                 # zh / en / th 字典（取自雛形 I 物件）
│   └── assets/                 # logo 等
├── data/                       # 執行期資料（uploads / recordings），不進版控
├── spec/                       # 本規格包
└── CLAUDE.md
```

### 1.3 可復用資源
| 資源 | 位置 | 用途 |
|---|---|---|
| UI 互動雛形 v6 | 既有 HTML | 對照畫面、互動、token、i18n 字典 |
| i18n 字典 | 雛形內 `I` 物件 | 直接搬入 web/i18n.js |

| Speakr 源碼 | 外部參考 | 批次轉錄流程「靈感」，不直接 fork |

## 3. User Stories

### Sprint 0：基礎

#### S-01 專案骨架與內網綁定
**對應：** NFR-4, NFR-7, SEC-9　**依賴：** 無　**被依賴：** S-02, S-03
**要做的事：** 建 FastAPI app、健康檢查端點；應用本體綁 `0.0.0.0`、Port 由 `APP_PORT` 指定（本機開發 3600／內網測試環境 3610）對內網開放；**其餘 3601–3699 保留本專案其他後端元件**；模型端點指向 LMS／vLLM／pod（建議綁 localhost）；FastAPI 靜態服務 web/（沿用 v6 HTML），API 走同源相對路徑；統一回應外型 `{data,error?,message?}`。
**驗收：**
- [ ] 應用本體（測試環境 3610）在內網其他機器以「主機內網 IP:3610」可達；模型端點（LMS/vLLM/pod）綁 localhost、不對內網外露
- [ ] `/api/health` 回 200；開根網址即載入 v6 介面
- [ ] 前端以相對路徑 `/api`、`/ws` 呼叫（無寫死 host）
**工時：** 4h

#### S-02 兩區儲存 + SQLite + 清除排程
**對應：** NFR-3, SEC-3, SEC-7　**設計：** §6 資料模型　**依賴：** S-01　**被依賴：** S-04, S-06, S-07
**要做的事：** 建 jobs/sessions/outputs/endpoints 表；實作 `/data/uploads`、`/data/recordings` 兩區，**檔案一律以伺服器產生 id 命名**並正規化路徑限制在 /data 內；7 天到期排程（依 expire_at 刪檔＋刪索引）。
**驗收：**
- [ ] 四張表建立、索引正確
- [ ] 寫入檔案落在正確 zone，路徑不含使用者原檔名
- [ ] 嘗試 `../` 之類路徑被擋
- [ ] 到期工作會連檔帶記錄被清除
**注意：** 原檔名只存 DB 供顯示，永不進入檔案系統路徑。
**工時：** 8h

#### S-03 模型路由 + 資源動態管理
**對應：** FR-21, FR-22, NFR-1, NFR-2, SEC-5　**設計：** §5 API-11, §2.6　**依賴：** S-01　**被依賴：** S-04, S-06, S-10
**要做的事：** endpoints CRUD（含 function 類型）；以 OpenAI 相容方式呼叫各端點；資源管理：啟動工作查可用量→reserve 上限定額（vLLM `gpu_memory_utilization` 設定）→閒置 N 分鐘 unload；上限納管 VRAM＋RAM＋儲存。
**驗收：**
- [ ] 可新增／停用端點並切換某功能的目標端點
- [ ] 啟動工作只取用定額、不超過 80% 上限
- [ ] 閒置逾時自動釋放顯卡，下次使用重載
- [ ] 取得不到資源時回 503 並標記需降級
**工時：** 12h

### Sprint 1：核心迴路

#### S-04 批次轉錄（前處理 + ASR + 切段）
**對應：** FR-12, FR-14　**設計：** §2.4, §5 API-01/03　**依賴：** S-02, S-03　**被依賴：** S-05, S-07
**要做的事：** API-01 接多檔上傳建立工作、序列處理；DeepFilterNet3 降噪→Silero VAD→**每 15 分鐘切段**→Qwen3-ASR 離線轉中文逐字稿；雙層進度（整體＋當前檔）寫入 DB 供 API-03 查詢。
**驗收：**
- [ ] 上傳 60 分鐘 m4a 切成 4 段、產出含時間戳中文逐字稿
- [ ] API-03 回正確的整體與當前檔進度
- [ ] 不支援格式被 400 擋（見 S-13）
**注意：** 單段勿超過模型 20 分鐘上限；切段邊界盡量落在靜音。
**工時：** 16h

#### S-05 批次翻譯（扇出多語）
**對應：** FR-13, FR-15　**設計：** §5 API-04, D-03　**依賴：** S-04　**被依賴：** S-08, S-09, S-12
**要做的事：** 對中文逐字稿，依 out_langs **一對多獨立翻譯**（不串接）；批次預設走 Gemma 4 31B Q8，可路由到 TranslateGemma／NLLB；結果存 outputs，供中／泰／英頁切換預覽。
**驗收：**
- [ ] 選中＋泰能各自產出獨立翻譯
- [ ] 切換目標模型後結果由對應端點產生
- [ ] 預覽可切語言頁與格式（文件／md／txt）
**工時：** 10h

#### S-06 即時 WebSocket 串流
**對應：** FR-1~FR-7, NFR-1　**設計：** §2.1/2.2, §5 API-06, SEC-6　**依賴：** S-03　**被依賴：** S-07
**要做的事：** `/ws/live` 接音框→VAD 邊界→Qwen3-ASR 串流（vLLM 後端）→即時翻譯（NLLB）；下行 partial/final＋多語譯文；同時持續錄音，停止時落地音檔＋逐字稿建立 session。WS 限來源、限訊息大小、閒置斷線。
**驗收：**
- [ ] 說中文後幾句內出現對應譯文
- [ ] interim 半透明、final 實心
- [ ] 停止後於錄音記錄出現該場、含音檔與逐字稿
- [ ] WS 僅接受內網來源（同網段）、拒絕外部
**注意：** 串流不回時間戳，時間以伺服器時鐘標記。
**工時：** 20h

## 4. 測試策略

### 4.1 範圍
| 類型 | 範圍 | 工具 |
|---|---|---|
| 單元 | services 商業邏輯（切段、扇出、路徑組裝、清除） | pytest |
| API | 每端點成功與失敗 case | pytest + httpx / Postman |
| 前端 | 表單驗證、i18n 切換、主題、資源預設隱藏 | 瀏覽器手動為主，必要時 Playwright 端對端 |
| 手動 | 完整流程走一遍 | §4.2 清單 |

### 4.2 手動測試清單
- [ ] 即時：開始→出中泰譯文→停止→錄音記錄出現該場
- [ ] 批次：多檔上傳→進度→結果中／泰預覽→匯出 docx
- [ ] 文件生成：選來源→切範本（同步）→匯出
- [ ] 設定：新增端點→切換某功能路由→生效
- [ ] 多語：切 en/th 全畫面框架字串一致、版面不爆
- [ ] 深色／淺色切換正常；資源量表預設隱藏、可開
- [ ] 邊界：超大檔被擋、斷網提示、特殊字元不破版

### 4.3 效能基準
- [ ] 批次 60 分鐘音檔可在合理時間內完成（記錄基準值）
- [ ] 即時譯文落後語句邊界約 1–3 句
- [ ] 閒置後顯卡確實釋放、再用能重載

### 4.4 資安檢核（對應 §5.2 SEC）
- [ ] SEC-1 模型／套件來源可信、雜湊比對、SCA 無高危 CVE
- [ ] SEC-2 上傳白名單＋MIME＋實際解碼；超限被擋
- [ ] SEC-3 路徑穿越（`../`、絕對路徑）被擋；檔名不入路徑
- [ ] SEC-4 逐字稿內含「指令字樣」不會被後處理 LLM 當指令執行
- [ ] SEC-5 單檔／佇列上限與動態預算生效
- [ ] SEC-6 WS 限來源／大小／閒置斷線
- [ ] SEC-7 到期確實刪檔＋索引；暫存清理
