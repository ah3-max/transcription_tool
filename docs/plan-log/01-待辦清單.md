## 🔵 進行中：S-06 即時 WebSocket 串流（主攻）
> 規格 FR-1~FR-7／設計 §2.1·2.2·API-06·SEC-6。plan：`Sprint1-S06-ws-streaming.md`，**動手前必讀該檔 §2.5 PoC 接線清單**（PoC 已踩過的雷，別重踩）。

- [ ] **ASR 串流客戶端 `services/asr.py`（新檔）**
  - 做什麼：用 `resolve_endpoint("asr")` 連 host vLLM 的 `/v1/realtime`（WebSocket），餵音框、收 partial/final delta。
  - 雷（PoC 實證）：握手序列 `session.update → commit(final=False) 啟動 → append → commit(final=True)`，**漏 `commit(final=False)` 會收不到任何 delta**；每個 delta 帶 `language {lang}<asr_text>` 前綴，**串流路要自己剝**（批次端點 server 端才會剝）。
  - 完成判準：丟一段中文音框能拿到去前綴的增量逐字稿。
- [ ] **WS 端點 `ws/live.py` → `/ws/live`（新檔，`ws/` 目前空的）**
  - 做什麼：上行音框 → VAD → ASR 串流 → 即時翻譯 → 下行 `{type:partial|final, src, translations:{th,en,…}, t}`。
  - 注意：**`t` 用伺服器時鐘**（串流不回時間戳，NG-6）；瀏覽器音訊要先降成 16kHz 單聲道 PCM16 再餵（PoC 餵的格式）。
  - 涉及：`main.py` 註冊 WS（1 行）。
  - 完成判準：說中文後幾句內前端出現原文＋譯文。
- [ ] **VAD／重疊切段**
  - 做什麼：用 Silero VAD 切句（先可用簡化版、保留掛點）。
  - 為什麼：固定 5s 緩衝會在段界切字（PoC 實測「長辈」被切成「长｜备」）→ VAD/重疊是必需、非選配。
- [ ] **★第 5 步：凍結 `sessions`/`outputs` 寫入契約**（先做，解 D 軌 S-07 依賴）
  - 做什麼：停止錄音時落地音檔（recordings 區）＋逐字稿，建立 `sessions`（name/langs/duration/status/audio_path/transcript_path）＋寫 `outputs`(kind=transcript/translation)，下行 `{type:saved, session_id}`。
  - 為什麼先做：此欄位契約是 S-07 後端的依賴，凍結後 D 軌可用假 session 並行開發＋pytest。
- [ ] **SEC-6 連線加固**：限同網段來源（Origin/來源 IP）、限連線數、限訊息大小、閒置斷線。
- [ ] **前端即時頁接線**（`web/app.js`）：interim 半透明（草稿）／final 實心（定稿）；麥克風授權、開始/停止。
