# 規劃：S-11 前端串接 v6 UI（原生 HTML，不導框架）

> 建立：2026-06-26 ｜ 類型：規劃 ｜ 配對開發日誌：`docs/dev_log/Sprint3-S11-frontend.md`（動工後建）
> 對應：FR-23/24、NFR-5/6、設計全部；依賴 S-04~S-10
> Plan B 可做性：🟡 大部分現在可做（接現有 API）；轉錄預覽/即時舞台待 S-04/S-06，先留 stub

## 1. 想解決什麼問題
把 v6 愛愛院版 HTML/CSS/JS 拆進 `web/`、由 FastAPI 靜態服務，以原生 `app.js` 串接全部 API（同源相對路徑 `/api`、`/ws`），讓五畫面能真的操作。

## 2. 要收斂的目標（驗收）
- 五畫面接真實 API、無 Vue/React/Vite 相依。
- 切語言（zh/en/th）全畫面框架字串一致、資源顯示**預設隱藏**。
- 內網其他機器以 IP:3610 可運作。
- 鍵盤可操作、對比 AA、內文 ≥19px／行高 1.8／可點 ≥48px。

## 3. 怎麼改、為什麼
- 拆 v6 HTML → `web/index.html` + `styles/tokens.css` + `app.js` + `i18n.js`（取雛形 `I` 物件）。
- `app.js` 串接：設定（`/api/endpoints` CRUD、`/api/resources`）、批次（上傳 `POST /api/jobs`、清單/進度 GET）、錄音記錄（`/api/sessions`，待 S-07）、文件生成（`/api/records`，S-08）、匯出（API-04，S-09）。
- 同源相對路徑、免烤 base URL（D-11）。
- 為什麼：內部工具、單人維護、雛形已含 i18n/主題/無障礙（D-11）。

## 4. 範圍邊界（做 / 不做）
- **做（現在可）**：設定頁（端點 CRUD/資源預設隱藏）、批次上傳＋清單＋進度、i18n/主題/無障礙、靜態服務拆檔。
- **待後續（先 stub/佔位）**：批次結果「逐字稿/翻譯預覽」（待 S-04/S-05 outputs）、即時舞台（S-06）、錄音記錄頁（S-07）、文件生成頁接 S-08、匯出接 S-09。

- [ ] 設定頁可新增/停用端點、看資源（預設隱藏）
