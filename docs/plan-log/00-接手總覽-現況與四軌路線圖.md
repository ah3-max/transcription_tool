## 0. 給接手者：30 秒讀懂

**vLLM-ASR PoC 已 GO**——Qwen3-ASR 在這張 Blackwell(sm_120) 上以 vLLM 跑得起來、批次與串流（`/v1/realtime`）皆可用。**但 app 端還沒接**：沒有 `services/asr.py`、`ws/` 是空的、上傳的 job 永遠停在 `queued`。接下來分四軌推進：

```
A 軌（獨立，進行中）   S-12 中→泰品質實測 ─────────────────────────┐  零交集
B 軌（近獨立，進行中） S-10 降級與錯誤處理 ──[凍結 降級訊號約定]──┐ │  解 C 軌依賴
C 軌（主攻）           ① NLLB 服務(:8001) ⟶ ② 完整 S-06 即時串流 ◀┘ │  接 PoC
                                              └─[凍結 sessions/outputs 契約]─┐
D 軌（跟在 C 後）       S-07 錄音記錄 ◀──────────────────────────────────────┘
```

**現在的第一個動作 → §3：把 NLLB 即時翻譯服務(:8001) 做出來並驗過**（C 軌的前置，PoC 只驗了 ASR、沒驗翻譯）。

## 3. 現在就做：C 軌第一步 — NLLB 即時翻譯服務(:8001)

> NLLB 是 C 軌與 A 軌的共同前置，且**目前沒有獨立 plan**（roadmap 四檔沒它），故在此展開為可執行步驟。依據 SOP §3.C ＋ PoC 的 sm_120 交叉教訓。做完請把版本/啟動指令回填 SOP §3.C，並視需要拆成獨立 dev_log。

NLLB 特性：純模型、**無內建 server、非 OpenAI 相容**（用 FLORES-200 語碼＋`forced_bos_token_id`）。語碼：中簡 `zho_Hans`、中繁 `zho_Hant`、英 `eng_Latn`、泰 `tha_Thai`。

1. **建 venv＋驗 sm_120（比照 PoC，別跳過）**
   ```bash
   uv venv ~/.venvs/nllb --python 3.12
   uv pip install --python ~/.venvs/nllb/bin/python torch transformers fastapi uvicorn sentencepiece
   # ★ 驗 sm_120：is_available()==True、get_arch_list() 含 'sm_120'、實跑一次 matmul 不報 no-kernel
   #   PoC 在 sm_120 過的是 cu130；若預設 torch 缺 sm_120 kernel → 改裝 cu130（--index-url .../whl/cu130）再驗
   ```
2. **最小服務 `~/services/nllb_server.py`（:8001）**：`POST /translate`，body `{text, src_lang=zho_Hans, tgt_lang}`，回 `{"data":{"translation":...}}`（對齊專案回應外型）。一對多扇出由 app 端對每個目標各打一次（D-03，不串接）。雛形見 SOP §3.C。
3. **冒煙**：`curl -s -X POST localhost:8001/translate -d '{"text":"早安","tgt_lang":"tha_Thai"}'` → 合理泰文；再用 1–2 句長照情境中文（含護理術語/中英夾雜）確認**中→泰直譯**可用。
4. **提速（建議正式採用）**：CTranslate2 int8，本地從官方權重轉（不引第三方權重，符合 SEC-1），VRAM ~3.4GB、延遲更低。
5. **接進 app**：`POST /api/endpoints` 註冊 `function=live_tr`、url=`http://host.docker.internal:8001`（**無 `/v1`**）；`.env` 的 `LIVE_TR_ENDPOINT` 已是此值。視需要建 systemd unit `stt-nllb`（`idle.py:30` 引用，CT2 int8 才適合常駐）。
6. **回填**：版本/啟動指令／sm_120 驗證結果寫回 SOP §3.C；更新本檔 §1.3 狀態。

**DoD**：`/translate` 中→泰直譯合理、VRAM 受框、app 容器經 `host.docker.internal:8001` 連得到、端點已註冊 `live_tr`。

## 4. 之後：完整 S-06（把 PoC 接進來）

細節在 `Sprint1-S06-ws-streaming.md`，**實作前務必讀該檔 §2.5 PoC 接線清單**（PoC 已踩過的雷，勿重新發現）。重點：

- **A** `/v1/realtime` 握手要 `commit(final=False)` 才會啟動 generation（漏了收不到 delta）。
- **B** ASR server 要以 `--hf-overrides '{"architectures":["Qwen3ASRRealtimeGeneration"]}'` 啟動才掛 `/v1/realtime` → 先建 `stt-vllm-asr` unit、可按需起停。
- **C** 串流輸出要剝 `language {lang}<asr_text>` 前綴（批次 server 端會剝、串流不剝）。
- **D** 固定 5s 緩衝會切字 → VAD/重疊切段是必需。
- **E** 瀏覽器音訊要降成 16kHz 單聲道 PCM16 再餵 realtime。
- **F** 不回時間戳、`t` 用伺服器時鐘；首 partial ≈5s。

S-06 第 5 步（sessions/outputs 寫入）一凍結，**D 軌 S-07** 後端即可並進。

## 6. 共用雷與開發紀律（所有軌通用）

- **共用 GPU**：殺 vLLM 要連 `VLLM::EngineCore` 子行程一起殺（`pkill -9 -f 'VLLM::EngineCore'`，父行程被 kill 後子行程會殘留佔 VRAM、cmdline 不含 vllm）；GPU 模型按需起＋閒置釋放（D-06）。
- **torch 版本**：sm_120 上用 **cu130**（非 cu128）；新 GPU venv 建好先驗 kernel。
- **app.js 多軌協調**：B（全域橫幅/toast）、C（即時頁）、D（記錄頁）動**不同區塊**，以「區塊不重疊」為界避免合併衝突。
- **資料模型同步**：改欄位/狀態 → 顯示／輸出／解析三處全鏈路同步；檔案一律 server id 命名，原檔名只存 DB（防穿越，D-07/SEC-3）。
- **紀律**：動手前先 grep/讀現況/`git status` 查證、引用真實行號；先寫規劃（`docs/plan-log/`）再動手、收尾寫 dev_log（`docs/dev_log/`）兩夾同名互 cross-ref；commit＝一個可驗收單元、引用 FR/S/D 編號；溝通與文件繁中。「寫好 ≠ 驗過」據實回報。
