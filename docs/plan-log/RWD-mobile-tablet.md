# 規劃：前端 RWD — 平板斷點補齊與三層級重構

> 建立日期：2026-06-26 ｜ 類型：規劃（事前）
> 對應：設計決策手冊 §1.2 導覽（sidebar→底部 tab bar）、§7.3 無障礙、D-11 原生前端
> 配對開發日誌：`docs/dev_log/RWD-mobile-tablet.md`
> 觸發：使用者檢視手機/平板瀏覽器（iPhone Safari / Chrome）優化，盤點後發現平板缺口

---

### 1. 想解決什麼問題
`web/index.html`（單檔、CSS 內嵌）目前只有 **2 層 RWD**：
- `@media (max-width:860px)`：手機層（側欄→底部 tab bar、單欄、且隱藏資源量/即時原文欄/表格端點欄/波形/場次 meta/qrow 進度條）。
- 其餘（≥861px）：桌機層（左側欄 228px ＋ 完整內容）。

**缺中間的平板層**，造成兩個問題：
1. **iPad 直向（744–834px CSS 寬，如 iPad mini 768、Air 820、Pro11 834）落入「手機層」**，被過度精簡——明明有空間，卻把資源量、即時原文欄、設定表格的「端點」欄、波形全隱藏，當成小手機對待。
2. **861–1024px 的觸控平板（多為 iPad 橫向）吃到桌機側欄**——觸控裝置用左側欄不如底欄好按。

### 2. 要收斂的目標（驗收標準）
建立 **三層級**斷點，且：
- 桌機（≥1025）：維持現狀（左側欄＋完整內容），**零變動**。
- 平板（601–1024）：底部 tab bar（觸控優先）＋單欄，但**保留較高資訊密度**（資源量、即時原文欄可開、表格端點欄、波形、場次 meta 全留）。
- 手機（≤600）：維持今日精簡行為，**不退化**。
- 對齊手機定位決策（見 §6）：手機/平板以**上傳音檔、檢視逐字稿、下載產出**為主，即時收音交給電腦端。

### 3. 要怎麼改、為什麼這樣
把單一 `@media (max-width:860px)` 拆成兩段、桌機優先級聯（沿用現有 px 與 desktop-first 結構，最小變動）：

- **`@media (max-width:1024px)`（版面層）**：側欄 `display:none`、`.app` 改單欄、顯示 `.mnav` 底欄、`.main` 留底欄淨空。→ 平板與手機共用。
- **`@media (max-width:600px)`（精簡層，置於 1024 之後）**：才隱藏資源量/原文欄/表格端點欄/波形/場次 meta/qrow 進度條、縮 padding 與 `--cap`、`.top` 收緊。→ 只有真手機才精簡。
- 連帶把第 258 行 `@media (max-width:860px){.top-divider/#resToggle…}` 改為 `max-width:600px`，讓平板段的資源量與其分隔線/眼睛切換鈕一起回來。

**斷點數值理由**：600 讓大手機橫向與小平板分流；1024 讓 iPad 直向（≤1024）歸平板用底欄、筆電/桌機（≥1025）用側欄；等同「直向→底欄、橫向大尺寸→側欄」的直覺切分。

**驗證級聯**：≤600 ∪ ≤1024（對 ≤600 視窗）≈ 原 ≤860 規則集（已逐條核對：`.main` padding 由 ≤600 的 `18px 16px 84px` 後寫覆蓋、`.top` 同理），故手機行為保持不變。

### 4. 範圍邊界（這次做 / 不做）
- **做**：平板層、三層級重構、`.top` 控制項斷點同步（兩處 `@media` 編輯，皆在 `<style>` 內）。
- **不做（另案）**：
  - `100vh → 100dvh`（iOS 網址列縮放 bug，1 行，待使用者點頭後做）。
  - 既有級聯死碼：`@media(max-width:460px){.ph:19px}` 被後方 `.ph{28px}` 蓋、手機 `--cap:21px` 被 `.stage-main{--cap:30px}` 蓋——**本次不動**（屬高齡大字覆寫的副作用，改它會變動現有觀感），僅標記。
  - `em` 相對斷點、`env(safe-area-inset-bottom)` 底欄安全區、iOS 錄音/串流平台能力實測（屬 D-16 手機策略另案）。
- **不碰**：JS、HTML 結構、其餘 CSS、後端。

### 5. 驗收檢查清單（完成逐項打勾）
- [ ] 桌機 ≥1025：左側欄＋完整內容與今日一致（側欄未消失、無底欄）
- [ ] 平板 601–1024：底部 tab bar 出現、無左側欄、單欄；**資源量/表格端點欄/波形/場次 meta 仍在**；即時原文欄可 toggle 出 2 欄
- [ ] 手機 ≤600：行為同今（精簡隱藏、底欄、字級間距縮）
- [ ] iPad 直向（768 / 834）落在平板層；iPhone 直向（≤430）落在手機層
- [ ] 三層皆無水平捲動；固定底欄不遮主內容（底部淨空足夠）

### 6. 決策與待確認
- **手機/平板定位（使用者本 session 拍板）**：即時**收音交給電腦端**（內網主機接麥克風），手機/平板負責**上傳已錄音檔轉逐字稿、檢視逐字稿、下載產出**。→ 正好避開 iOS Safari「背景/鎖屏中止錄音」（WebKit #198277）與 WebSocket 背景斷線。**建議補入設計決策手冊為 D-16（手機策略），並在開發執行手冊加 iOS Safari 檢核清單**（本次不寫 spec，另案）。
- 斷點值 600 / 1024 為可調預設；平板維持「底欄」而非還原側欄，理由為觸控優先（可改）。
- 可視化對比（桌機/平板/手機三欄）另以 Artifact 產出，對齊本三層級設計。
