## 2026-06-26　備素材＋評測腳本

**原本怎麼樣**
- `server/services/translate.py` 的扇出與系統提示已可用，但無評測素材與跑批工具。

**發生什麼／要做什麼**
- 規劃收標準需「≥3 模型 × ≥5 段」量化比較＋母語者評語。母語者評分非我可代勞，
  故本回合先把可重複跑的素材＋腳本＋評分表結構備齊，跑批/評分交使用者。

**改了什麼**（皆在 `docs/eval/s12/`，**不進正式碼路徑**）
- `segments.json`：8 段真實長照情境中文（護理交班/給藥/跌倒/吞嚥/壓瘡/排程/情緒/輸出入量；
  含護理術語與中英夾雜與口語），來源固定 zh、目標 th。
- `run_eval.py`：一次性腳本，對每段打各候選模型 `/chat/completions`，系統提示與正式
  `translate.py` 的 `_system_prompt('th')` 一致；輸出 `results-*.md`（泰譯對照）＋
  `scores-*.csv`（母語者填準確/流暢/術語 1–5＋評語），時間戳命名不覆蓋。支援 `--base`/`--model`。
- `README.md`：候選模型、跑批指令、評分維度（收標準）、收斂為結論的步驟與待辦。

**遇到什麼困難／又怎麼改**
- 規劃原列 NLLB-3.3B；host 實際提供的是 Typhoon-Translate-4B（泰語特化），已在 README 註記改以實際三方比較。
- 模型 `base_url`/`model` 名稱依執行器而異 → 腳本頂部 `MODELS` 可改、並提供 `--base`/`--model` 覆蓋。

**最後如何解決**
